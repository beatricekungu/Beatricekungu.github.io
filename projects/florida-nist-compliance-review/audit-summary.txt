# Florida Public-Sector NIST Compliance Review Summary

## Executive Summary

This project reviewed public Florida cybersecurity audit themes and translated them into a NIST-aligned GRC control review. The purpose was to demonstrate how public audit findings can be organized into control areas, evaluated for compliance status, assigned risk levels, and converted into practical remediation actions.

The review identified several common cybersecurity governance themes, including access control, privileged access, incident response, audit logging, security awareness, policy governance, vulnerability management, and contingency planning.

## Key Findings

### 1. Access Control Requires Strong Review Evidence

Access management should include documented approvals, recurring reviews, and timely removal of unnecessary access. Without consistent review evidence, an organization may struggle to prove that access is appropriate and current.

### 2. Privileged Access Requires Additional Oversight

Privileged access presents a higher risk because elevated permissions can create significant exposure if they are not properly approved, reviewed, and monitored.

### 3. Incident Response Should Include Clear Escalation and Recovery Steps

Incident response processes should include severity levels, escalation paths, roles, responsibilities, reporting requirements, and closure documentation. These elements help organizations respond consistently during cybersecurity events.

### 4. Audit Logging Needs Documented Review Activity

Collecting logs is not enough for audit readiness. Organizations should be able to show that logs are reviewed on a defined schedule and that review results are documented.

### 5. Policy Governance Should Include Ownership and Review Dates

Cybersecurity policies should include owners, review cadence, approval history, version control, and last-reviewed dates. This supports governance maturity and audit readiness.

### 6. Vulnerability Management Should Track Remediation Through Closure

Vulnerability findings should be prioritized, assigned, tracked, and closed with evidence. Without tracking, high-risk findings may remain unresolved.

## Recommended Next Steps

1. Create a centralized compliance evidence library.
2. Assign control owners for each review area.
3. Add review dates, approval history, and version control to governance documents.
4. Create recurring access, log, policy, and vulnerability review schedules.
5. Use a remediation tracker to monitor findings through closure.
6. Document escalation paths for cybersecurity incidents.
7. Retain evidence for audit readiness and leadership review.

## Business Impact

This project shows how public cybersecurity audit themes can be translated into practical GRC actions. It demonstrates how structured documentation can help leadership understand risk, assign ownership, and improve audit readiness.

## Project Note

This is a public-source portfolio project. It does not claim to be a full internal audit of a Florida agency or private company. The purpose is to demonstrate control mapping, evidence review, compliance status assignment, risk rating, and remediation planning using public cybersecurity audit sources and NIST-aligned concepts.
