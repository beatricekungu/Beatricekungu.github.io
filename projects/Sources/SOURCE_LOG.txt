# Source Log

## Purpose

This source log documents the public materials used to support the Florida Public-Sector NIST Compliance Review project.

The purpose of this file is to show the evidence trail behind the project. These sources support the control mapping, audit themes, NIST alignment, compliance review logic, risk rating, and remediation recommendations.

## NIST Sources

| Source | Link | How I Used It |
|---|---|---|
| NIST Cybersecurity Framework 2.0 | https://csrc.nist.gov/pubs/cswp/29/the-nist-cybersecurity-framework-csf-20/final | Used as the primary cybersecurity framework for organizing governance, identify, protect, detect, respond, and recover themes. |
| NIST SP 800-53 Rev. 5 | https://csrc.nist.gov/pubs/sp/800/53/r5/final | Used as the control catalog for mapping audit themes to security and privacy control areas. |
| NIST SP 800-53A Rev. 5 | https://csrc.nist.gov/pubs/sp/800/53/a/r5/final | Used to understand how security and privacy controls can be assessed. |
| NIST SP 800-61 Rev. 3 | https://csrc.nist.gov/pubs/sp/800/61/r3/final | Used to support incident response, reporting, escalation, and recovery review areas. |
| NIST SP 800-30 Rev. 1 | https://csrc.nist.gov/pubs/sp/800/30/r1/final | Used to support risk assessment logic and risk-level assignment. |

## Project Note

This project does not claim to be a full internal audit of a Florida agency or private company.

This is a public-source GRC portfolio project that uses publicly available cybersecurity audit materials and NIST publications to demonstrate control mapping, evidence review, compliance status assignment, risk rating, and remediation planning.
