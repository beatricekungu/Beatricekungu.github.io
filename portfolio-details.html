Globalmed Enterprise Security PoC

Azure SQL Dynamic Data Masking (DDM) Implementation

🎯 Executive Summary

Demonstrating the implementation of Dynamic Data Masking (DDM) within Azure SQL Database to enforce the principle of least privilege. This PoC protects Personally Identifiable Information (PII) and Protected Health Information (PHI) from unauthorized internal viewing without altering the underlying data layer.

🎛️ Masking Architecture

  • HIPAA Standard: Names masked to first letter.
  • Contact Privacy: Native email() function applied.
  • Identity Protection: SSN replaced with default string.
  • PCI-DSS Compliance: Credit Cards exposed to last 4 digits only.

⚙️ Phase 1: Schema Initialization & Masking

The Globalmed table was engineered with masking functions natively attached to the column schema, ensuring that data protection is enforced at the database engine level, regardless of how the application queries it.

Fig 1: Successful deployment of the Globalmed schema with embedded DDM rules.

🔒 Phase 2: Role-Based Access Validation

To validate the security controls, distinct enterprise roles were created. The HelpDeskSupport profile was granted standard read access, while the ChiefComplianceOfficer profile was explicitly granted the UNMASK permission.

Fig 2: Query results demonstrating the dynamic application of masking based on user execution context.
